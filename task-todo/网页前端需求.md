# 网页前端需求

## 先读这个

开始任何前端改造之前，先完整阅读 `chrome插件` 文件夹里的资料。

上传到 GitHub 之后，这一批资料会放在：

- `前端/`

这里面不是普通备份，而是我们之前用 Chrome 插件做前端测试时留下来的关键上下文，必须先看：

- 插件源码
- 插件导出的抓取结果
- 过程记录
- 协作说明
- 当前进度交接
- 聊天框控制器方案

这些内容能帮助后续的人理解：

1. 我们之前具体测过哪些前端页面
2. Chrome 插件是怎么抓页面和事件的
3. 之前的测试结果长什么样
4. 哪些方案试过，哪些问题已经暴露过
5. 后续前端还原、抓取、验证时应该优先参考什么

## 强调要求

不要跳过 `chrome插件` 这一整包资料，直接开始写前端。

必须先阅读 `前端/` 目录中的信息，再开始：

- 前端页面还原
- 考试页面交互复刻
- 抓取链路验证
- 数据结构对照
- dashboard / review / directions 相关页面工作

## 当前这批资料的用途

`前端/` 目录里的资料主要用于：

- 还原之前的测试上下文
- 对照 Chrome 插件抓到的页面和行为
- 帮助理解我们过去做过的前端实验
- 作为后续 PDF / JSON / 前端壳子联调时的参考材料

## 建议阅读顺序

1. `当前进度交接.md`
2. `协作分工说明.md`
3. `聊天框控制器方案.md`
4. 两个 `ap-capture-*` 导出目录
5. `ap-exam-capture-v11/` 插件源码

## 与当前项目的关系

这批资料不是正式网站运行目录的一部分，但它们对前端工作非常重要。

它们的定位是：

- 前端测试资料
- Chrome 插件抓取实验资料
- 历史验证过程记录

所以后续凡是做前端相关工作，都应先参考：

- `前端/`
- `task-todo/网页前端需求.md`

## 目标

让后续接手的人看到 GitHub 仓库后，能立刻知道：

- 前端测试资料放在哪里
- 哪些文件必须先看
- 为什么这些资料有参考价值
